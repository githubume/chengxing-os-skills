# habit-rebuild 示例：睡前刷手机

本示例完全虚构，不来自真实用户。

## 用户请求

> 我工作日上床后会刷短视频四十分钟到一小时，最近七天有五天如此。加班后更严重。我试过应用限额，但三天后会自己关掉。希望先试七天，每天只记两项。

## 预期触发

- 插件：`habit-rebuild`
- 工作流：`behavior-assessment`，随后 `seven-day-plan`
- 路由：B1；若后续出现持续严重失眠、危险驾驶、自伤、药物误用等信号，应重新路由并停止普通计划。

## 行为单元

| 项目 | 当前信息 |
|---|---|
| 背景状态 | 工作日晚上；加班后更疲惫 |
| 直接前因 | 完成洗漱并带手机上床 |
| 可观察动作 | 在床上连续刷短视频 40–60 分钟 |
| 即时结果 | 快速放松，暂时不再处理工作感受 |
| 延迟代价 | 入睡推迟，第二天更疲惫 |
| 未知 | 手机不在卧室时是否会转向其他延迟行为 |

## 竞争假设

1. 疲惫时短视频承担快速放松功能（中置信度）。
2. 手机随手可得和上床动作已形成稳定提示（中置信度）。

首轮只验证第二项：先改变手机位置，同时保留一个低成本放松替代。

## 七天最小实验

共同规则：完成洗漱后把手机放在卧室外充电；上床后播放提前下载的十分钟音频。七天保持同一规则，不每天换方案。

| 要素 | 设计 |
|---|---|
| 触发 | 完成洗漱 |
| 最低动作 | 把手机接到卧室外充电线 |
| 标准动作 | 上床后只播放十分钟离线音频 |
| 环境 | 充电线移到客厅；床边放纸质读物 |
| 反馈 | 第二天早晨在纸卡上打勾 |
| 指标 1 | 上床后手机使用分钟 |
| 指标 2 | 完成环境动作的天数 |

恢复脚本：如果已经拿回手机，在当前视频结束时放回；不追补、不归零，第二天继续共同规则。

## 跟踪

- 第 3 天：环境动作是否真正执行？替代放松是否足够容易？
- 第 7 天：执行度和手机使用时长是否分别变化？是否出现其他延迟行为？

## 不应出现

- “你就是缺乏自律”；
- “七天一定戒掉手机”；
- 把普通刷手机直接诊断为成瘾、ADHD 或抑郁；
- 未经同意保存、上传或发送记录。
