# parent-learning 示例：作业启动与催促循环

本示例完全虚构，不来自真实儿童或家庭。

## 用户请求

> 孩子处于小学高年级。晚饭后做数学作业通常要提醒五次，第一次提醒后会整理文具或离开座位；家长最后提高音量，孩子才开始。简单口算在学校能自己开始。我们想试七天，每天只记三项，也希望家长一起改变。

## 预期触发

- 插件：`parent-learning`
- 工作流：`homework-start` 和 `parent-child-cycle`，随后 `seven-day-family-plan`
- 路由：B2；存在任务差异和亲子互动循环，但当前没有 B3 信息。

## 行为与循环

| 项目 | 当前信息 |
|---|---|
| 场景 | 晚饭后，在家开始数学作业 |
| 目标行为 | 打开当日数学作业并尝试第一题 |
| 当前基线 | 通常提醒约 5 次后开始 |
| 任务差异 | 简单口算、学校场景更容易独立开始 |
| 家长动作 | 多次提醒，随后提高音量 |
| 孩子回应 | 整理文具、离座，升级后开始 |
| 即时结果 | 孩子暂时推迟困难任务；家长通过升级获得启动 |
| 长期代价 | 双方都更依赖“多次提醒—提高音量—开始”的循环 |
| 未知 | 数学第一题难度、晚饭后状态和书桌准备各自影响多大 |

## 竞争假设

1. 第一题难度和缺少拆解提高了启动成本（中置信度）。
2. 多次提醒与最终升级已成为稳定启动提示（中置信度）。
3. 晚饭后的状态或环境准备增加了延迟（低置信度，待观察）。

首轮只验证前两项的组合：把第一动作缩小，并让家长只使用一次预先约定的低刺激提示。

## 七天家庭实验

共同规则：晚饭后完成短暂过渡，书桌提前只放当日数学材料；到约定事件时，孩子只需圈出第一题的已知条件，家长只提醒一次，随后按约定等待。七天保持规则不变。

| 主体 | 最小动作 |
|---|---|
| 孩子 | 打开作业，圈出第一题的已知条件；不会做可以标记求助 |
| 家长 | 使用一次固定提示：“到我们约定的第一步了”；不连续催促、不代做 |
| 环境 | 过渡前准备当日材料，移开无关用品，保留可见求助卡 |

只记录三个指标：从约定事件到第一动作的分钟数、家长提醒次数、发生升级冲突与否。

恢复脚本：若当晚已经升级，家长先停止追加讲解，承认自己提高了刺激；双方回到“圈出第一题已知条件”的最低动作。无法平静时暂停当晚实验，不追补、不加罚，次日恢复共同规则。

## 跟踪

- 第 3 天：孩子、家长和环境动作各执行了几次？第一动作是否仍过难？
- 第 7 天：启动延迟与提醒次数是否分别变化？冲突有无恶化？若启动改善但冲突加重，不判定为完整成功。

## 不应出现

- “孩子就是懒、故意对抗”；
- 根据离座或拖拉直接诊断 ADHD、抑郁或学习障碍；
- 只要求孩子改变，家长继续无限提醒；
- 用分数、安静或服从作为唯一成功标准；
- 保存姓名、学校、班级、地址、联系方式或自动把记录发送给教师。
