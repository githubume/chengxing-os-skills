# 作业启动分析

## 必须先读

读取 `../../references/chengxing-shared/safety/routing.md`、`../../references/chengxing-shared/core-models/behavior-method.md` 和 `../../references/chengxing-shared/quality-gates/output-quality.md`。先做 B3；持续明显躯体/睡眠异常、严重拒学、自伤、暴力或无法保持安全时停止普通启动方案。

## 定位行为链

按顺序检查：状态准备→进入学习场景→启动→维持→困难应对→完成→订正→收尾→中断恢复→独立迁移。

## 执行

1. 把“磨蹭、不自律、不听话”改写为任务、时间、第一动作、启动延迟和最近实例。
2. 比较简单/困难任务、不同学科、学校/家庭、独立/家长在场和顺利例外。
3. 还原家长在启动前后做什么，不把家长动作当作背景噪声。
4. 形成最多 3 个竞争假设：能力/拆解、状态、环境、任务情绪、提示、关系或后果。
5. 只选择一个首轮瓶颈；孩子动作必须小到可独立尝试。
6. 同时定义一个家长动作和一个环境动作；不得只要求孩子改变。

## 诊断诱因

用户要求把拖拉、离座或分心确诊为 ADHD、抑郁、学习障碍等时，只说明当前行为不足以支持诊断，转回可观察差异和简短专业评估边界。不列诊断标准、量表、专科路径或外部指南链接。

## 输出

1. B1/B2 路由和行为重定义；
2. 学习行为链中的卡点；
3. 已知、未知和最多 3 个假设；
4. 孩子第一动作、家长支持动作、环境动作；
5. 今晚 1 个低风险试验；
6. 最多 3 个指标，从启动延迟、提醒次数、中断、恢复或独立比例中选择；
7. 失败恢复、停止和升级条件。

不以服从、安静或分数作为唯一成功标准，不使用比较、羞辱、代做或剥夺基本需要。
