# 家庭学习实验跟踪

## 必须先读

读取 `../../references/chengxing-shared/safety/routing.md`、`../../references/chengxing-shared/core-models/behavior-method.md`、`../../references/chengxing-shared/case-schema/case.schema.json` 和 `../../references/chengxing-shared/quality-gates/output-quality.md`。新信息出现儿童安全或其他 B3 信号时，立即停止普通复盘，先执行安全分流。

## 输入处理

优先使用本轮用户提供的记录。只有负责成人已同意本地保存并明确要求使用时，才读取本地案例。记录为空、口径改变、只有印象或只有分数时明确局限，不把缺失当失败，不伪造趋势。

## 复盘

1. 分开描述孩子执行度、家长执行度和环境调整完成度。
2. 比较目标行为变化、恢复速度及关系/情绪副作用；孩子表现改善但冲突加重时，不判定为完整成功。
3. 检查成绩变化是否能归因；没有足够周期或对照时，只作背景观察。
4. 判断首轮假设获得、失去或尚无足够支持。
5. 检查动作是否太难、家长提示是否升级、环境是否真正改变、指标是否敏感或出现替代性问题。
6. 只选择保留、简化、替换、暂停、升级或继续收集信息之一；下一版仍优先一个关键变量。

## 输出

- 数据时间范围、完整性和口径；
- 孩子、家长、环境执行度三栏；
- 行为变化、恢复速度、关系副作用三栏；
- 假设更新、证据与不可判断项；
- 一项决策及理由；
- 下一版孩子、家长、环境各一个最小动作；
- 恢复脚本、下次跟踪点和停止/升级条件。

如用户要求更新本地案例，先展示更新后的完整 JSON 预览并再次确认，再用 `local_store.py case write` 原子覆盖。不得虚构第 7/14/30 天记录，也不得把孩子单方面顺从写成家庭系统改善。
