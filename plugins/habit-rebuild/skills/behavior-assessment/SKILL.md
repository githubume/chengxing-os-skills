# 习惯行为评估

## 必须先读

读取：

1. `../../references/chengxing-shared/safety/routing.md`
2. `../../references/chengxing-shared/core-models/behavior-method.md`
3. `../../references/chengxing-shared/quality-gates/output-quality.md`

先按 B3 路由；B3 时停止本流程。主要问题是孩子学习时重定向到 `parent-learning`；主要目标是关系沟通时重定向到 `conflict-reset`。

## 诊断诱因优先分支

若用户要求“正式确诊”“是不是 ADHD/抑郁/成瘾/强迫”等疾病结论，优先于普通评估执行本分支：

1. 用一句话说明单一行为和当前信息不能支持诊断，本插件不提供诊断。
2. 不列诊断标准、症状清单、筛查问卷、量表阈值、专科路径、国家/地区指南或任何外部链接。
3. 只把问题转回最近一次具体行为、场景、启动延迟、影响和例外成功；最多建议记录 1–3 个行为指标。
4. 只有当用户描述持续明显功能影响时，用一句话建议寻求当地合格专业人员的个体化评估，不指定疾病结论。
5. 回答到此结束；除非用户转而要求行为分析，不展开七天计划。

## 执行

1. 把“没自律、懒、控制不住”改写为时间、地点、动作、频率/时长和替代行为。
2. 信息不足时只问 1–3 个能区分机制的问题；优先最近一次、直接前因、即时收益和例外成功。
3. 形成“背景状态→直接前因→动作→即时结果→延迟代价→下次预期”。
4. 用九个诊断镜头扫描，但只展示与当前判断有关的变量。
5. 生成最多 3 个竞争假设；为每个写支持观察、反驳观察、下一测试和置信度。
6. 按路由规则选择 B1 或 B2；只选一个首要瓶颈。
7. B2 在缺少决定性基线时，先提出一个澄清问题或给出条件分支，不擅自选择精确健康阈值。
8. 若用户已同意本地画像，可读取画像补充背景；画像与当前描述冲突时以当前信息为准并提示更新，不静默改写。

## 输出

按以下顺序：

1. 范围与 B1/B2 路由理由；
2. 一句话行为重定义；
3. 行为循环表；
4. 已知、未知和待验证信息；
5. 最多 3 个竞争假设；
6. 首要瓶颈与为什么先处理它；
7. 一个今天可做的最小观察或动作；
8. 最多 3 个基线指标；
9. 失败恢复和停止/升级条件；
10. 下一步进入 `seven-day-plan` 所需的信息。

不要在评估阶段承诺计划有效，也不要把普通拖延直接解释为 ADHD、抑郁、成瘾、意志力不足或“多巴胺坏掉”。当用户要求诊断时，只说明当前信息不能支持诊断，给出行为观察路径和简短专业评估边界；除非用户另行要求且来源已核验，不列疾病诊断标准、量表或指南链接。
