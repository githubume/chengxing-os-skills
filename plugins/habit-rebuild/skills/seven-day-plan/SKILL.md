# 七天习惯实验

## 必须先读

读取 `../../references/chengxing-shared/safety/routing.md`、`../../references/chengxing-shared/core-models/behavior-method.md` 和 `../../references/chengxing-shared/quality-gates/output-quality.md`。B3 时不生成计划。

## 前置条件

至少确认：一个目标行为、一个常见场景、一个替代行为、大致基线和过去尝试。缺失时只问最有价值的 1–3 项。多因素或反复失败时标记 B2，但仍只设计一个首轮验证。

## 设计

1. 选一个首要瓶颈和一个竞争假设。
2. 把动作缩到最差状态下也可能完成的最低版本，默认 2–10 分钟或一个明确动作。
3. 使用事件型触发：“当 X 发生，我先做 Y”；作息不固定时不用固定钟点硬套。
4. 预先调整一个环境变量，并为高即时奖励行为保留合理替代放松。
5. 设计即时、非羞辱反馈。
6. 选择最多 3 个指标：启动延迟、频次/时长、完成天数、恢复时间或主观难度。
7. 写出漏做当日和连续中断后的最低恢复动作，不追补、不归零。
8. 设置第 3 天和第 7 天检查，以及停止/升级条件。
9. 咖啡因、睡眠、饮食和运动等健康相关变量只在已知基线基础上做小幅可逆调整；不引用未经核验的统一时间、剂量或强度阈值。

## 输出

- 路由、目标和本轮假设；
- 七天共同规则，而不是七套完全不同任务；
- “触发—动作—环境—反馈—记录”表；
- 最低版本、标准版本和可选加码；
- 最多 3 个指标与首日基线；
- 恢复脚本；
- 第 3/7 天复盘问题；
- 停止和专业升级条件。

不得同时要求超过 3 个新动作，不使用“21 天一定形成习惯”“彻底戒掉”或结果保证。
