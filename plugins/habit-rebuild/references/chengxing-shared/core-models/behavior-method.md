# 成长行为学运行方法

## 1. 输出对象

把“懒、没自律、不听话、情绪化”等评价改写成一个场景行为单元：

```text
背景状态 → 直接前因 → 可观察动作 → 即时结果 → 延迟代价 → 下次预期
```

只把用户明确陈述的内容写成事实。解释行为功能时使用“可能、待验证、当前假设”，并写出什么观察会支持或反驳它。

## 2. 最小信息

优先获得：

1. 谁在什么时间、地点和任务中做了什么；
2. 最近一次实例，以及频率、时长或启动延迟；
3. 行为前刚发生了什么；
4. 行为后立刻获得或避免了什么；
5. 用户希望出现的替代行为；
6. 过去尝试及例外成功。

一次只问 1–3 个最能改变方案的问题。用户拒绝回答时，给出明确标注假设的低风险通用观察，不虚构事实。

## 3. 九个诊断镜头

将下列内容当作观察镜头，而不是评分或诊断：

| 镜头 | 要问的问题 |
|---|---|
| 目标 | 行为是否具体、标准是否清楚？ |
| 能力 | 是否会做、能否拆解、是否知道如何求助？ |
| 状态 | 睡眠、疲劳、饥饿、身体不适或情绪是否改变了难度？ |
| 机会与环境 | 时间、空间、工具、干扰和资源如何影响行为？ |
| 自主与意义 | 用户是否理解理由并拥有合理选择？ |
| 提示 | 行为前是否有稳定、清晰、及时的线索？ |
| 关系支持 | 他人的提醒、控制、示范或协作如何影响循环？ |
| 预期后果 | 行为立刻得到什么，延迟付出什么？ |
| 行为历史 | 过去重复形成了什么预期、例外和复发模式？ |

## 4. 功能假设

常见功能包括：逃避困难、获得刺激、调节情绪、恢复控制、获得关注、节省时间、降低不确定性和社会认同。不要只看动作表面；替代行为需要以更低或相近成本承担相似功能。

每次最多保留 3 个竞争假设，并为每个假设写：

- 支持它的已知观察；
- 与它不一致的观察；
- 下一次最小验证办法；
- 当前置信度（低/中/高），且高置信度也不等于诊断。

## 5. 阶段与瓶颈

先定位问题主要发生在：准备、进入场景、启动、维持、困难应对、完成、恢复或迁移。再从可改变性、影响大小和风险中选择一个首要瓶颈。

不要为了“全面”同时改变多个变量。B1 默认一个关键变量；B2 也只先验证一个竞争假设。

## 6. 最小充分干预

一个实验至少说明：

- 触发：何时开始；
- 动作：第一步是什么；
- 环境：提前准备或移除什么；
- 反馈：如何尽快看见完成；
- 责任：谁做什么；
- 指标：最多 3 个；
- 恢复：漏做后的下一最小动作；
- 停止：出现什么风险或副作用就暂停。

计划不是处方。优先可逆、低成本、低风险、可在 3–7 天内获得新信息的实验。

涉及咖啡因、睡眠时点、饮食、药物、运动强度或其他健康相关变量时，不给所有人统一的生理阈值。先取得用户基线与限制，只做基线上的小幅可逆调整；需要个体化健康判断时升级给合格专业人员。

## 7. 跟踪更新

跟踪时分开评价：

1. 是否执行了实验；
2. 目标行为是否改变；
3. 是否出现副作用；
4. 哪个假设获得或失去支持；
5. 下一版保留、简化、替换还是升级。

默认第 3 天检查执行障碍，第 7 天评估变化；需要维持时再看第 14/30 天。一次失败不归零，不虚构未发生的跟踪结果。
