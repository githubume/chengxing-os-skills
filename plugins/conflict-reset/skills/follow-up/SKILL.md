# 冲突复位跟踪

## 必须先读

读取 `../../references/chengxing-shared/safety/routing.md`、`../../references/chengxing-shared/core-models/behavior-method.md`、`../../references/chengxing-shared/case-schema/case.schema.json` 和 `../../references/chengxing-shared/quality-gates/output-quality.md`。新出现暴力、胁迫、跟踪、报复、自伤或他伤等 B3 信号时停止普通复盘，不建议继续沟通实验。

## 输入处理

优先使用用户提供的记录。只有用户已同意本地保存并明确要求使用时，才读取本地案例。缺失记录不等于未执行；对方陈述不得由用户或模型代填。

## 复盘

1. 分开评价暂停/边界是否执行、冲突强度和频率、恢复时间、修复行为及副作用。
2. 检查权力和安全是否变化；表面更安静但恐惧、控制或报复增加时，不判定成功。
3. 谈话平静但同类问题持续重复时，转查规则、分工、资源或执行条件，而不是继续润色话术。
4. 判断首轮假设获得、失去或没有足够支持。
5. 只选择保留、简化、替换、暂停、升级或继续收集信息之一。

## 输出

- 数据时间范围、完整性和单方陈述边界；
- 过程执行、行为变化、安全/关系副作用三栏；
- 假设更新与不可判断项；
- 一项决策和下一版最小动作；
- 失败恢复、下次跟踪点和停止/升级条件。

如用户要求更新本地案例，先显示完整 JSON 预览并再次确认，再用 `local_store.py case write` 原子覆盖。不得虚构对方反馈、第 7/14/30 天结果或关系改善。
