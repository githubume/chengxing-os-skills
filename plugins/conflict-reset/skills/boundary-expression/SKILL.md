# 边界表达

## 必须先读

读取 `../../references/chengxing-shared/safety/routing.md`、`../../references/chengxing-shared/core-models/behavior-method.md` 和 `../../references/chengxing-shared/quality-gates/output-quality.md`。先检查暴力、胁迫、跟踪、经济/证件控制和报复风险；符合 B3 时不建议直接发送边界声明或告知求助安排，改走安全路由。

## 结构

```text
当[可核查行为]发生时，[具体影响]。
我请求[一个可执行请求]。
如果继续，我会[本人可执行、合法、成比例的保护行动]。
```

1. 区分偏好、请求、边界和威胁；边界核心是用户自己会做什么。
2. 不控制对方思想、情绪、社交、工作、金钱或基本资源。
3. 行动必须可兑现、成比例且不依赖恐吓；职场权力差异中考虑留痕、正式渠道和报复风险，但不虚构政策。
4. 若边界已多次表达仍被突破，分析执行条件，不无限升级措辞。

## 输出

- B1/B2 与权力/安全说明；
- 一句简短版和一段完整版本；
- 请求与本人行动的区别；
- 合适渠道和发送时机；
- 对方争辩、拒绝或报复时的退出/记录动作；
- 最多 3 个指标、第 3/7 天复盘和升级条件。

不得写“让对方害怕、不敢反驳”的话，不威胁曝光、羞辱、剥夺资源或非法报复，不把控制对方包装成边界。
