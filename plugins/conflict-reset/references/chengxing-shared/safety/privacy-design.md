# 本地画像与隐私设计规则

> 本文件是成行 OS 自身的产品安全基线。它只约束插件如何少收、少存和让用户控制本地数据，不依赖或调用其他领域 Skill。

## 1. 默认状态

- 默认不保存会话和画像。
- 默认不联网、不上传、不遥测、不联系第三方。
- 插件可以在内存中形成画像草稿；写入前必须展示字段、用途和路径。
- 只有用户明确同意当前写入后才落盘；“继续聊天”不等于同意保存。
- 同意按“本地画像、案例导出、匿名研究”分别记录，不捆绑。

## 2. 数据最少化

允许优先保存：年龄段、角色、目标行为、行为基线、场景、用户选择的偏好、实验版本和跟踪摘要。

默认不保存：真实姓名、生日、学校/班级、单位、地址、证件、电话、邮箱、精确位置、原始聊天、第三方敏感信息、医疗文书、危机细节和账号标识。

儿童画像仅使用年龄段或年级段；示例和测试全部虚构。第三方没有同意时，只记录用户自己的可观察经历，不把对第三方动机、疾病或隐私的推测写成事实。

## 3. 写入流程

固定顺序：

```text
收集最少信息 → 生成去标识化预览 → 显示用途和本地路径
→ 用户明确确认 → 原子写入 → 返回写入摘要和删除方法
```

用户修改或撤回时，更新同一记录并保留最少审计字段。不得把撤回后的内容留在“备份”“案例候选”或日志中，除非用户另有明确选择且产品已经说明保留规则。

## 4. 本地目录

默认根目录：`~/.chengxing-os/`。插件安装目录只放公开模板，绝不放用户画像。

```text
~/.chengxing-os/
├── person-profile.json
├── habit-rebuild/profile.json
├── parent-learning/profile.json
├── conflict-reset/profile.json
└── cases/<case-id>/case.json
```

写入必须使用路径约束和原子替换；不得跟随逃逸根目录的软链接，不得覆盖用户未选择的文件。升级、卸载插件默认不删除画像；单独的删除操作必须列出将删除的路径并取得确认。

## 5. 导出

导出前必须：

1. 询问接收目的和接收方类型；
2. 只选择必要字段；
3. 移除姓名、学校、单位、地址、联系方式和无关第三方信息；
4. 区分观察事实、用户解释、模型假设和未验证结论；
5. 展示完整预览；
6. 取得对本次文件写入的明确确认。

插件只生成本地文件，不自动发送。用户要求外发时，说明当前插件不代为联系第三方，并提醒用户再次核对接收方和内容。

## 6. B3 数据

B3 的详细危机信息默认不写入画像或案例。为继续安全对话可以在当前上下文中使用最少信息；如用户主动要求记录，仍只保存不含手段细节的最小摘要，并再次提示本地文件可能被同设备使用者看到。

涉及胁迫控制或跟踪时，不建议在可能被监控的设备留下显眼记录；不自动创建文件。

## 7. 用户控制

每个插件必须支持自然语言请求：

- “显示你准备保存什么”；
- “查看我的本地画像”；
- “修改这项信息”；
- “导出我的记录”；
- “删除这个插件的画像”；
- “删除全部成行 OS 本地数据”。

任何删除操作先 dry-run 列表、再确认、后执行；不得删除根目录外内容。

## 8. 发布前隐私门禁

在运营主体、服务入口和真实数据流确定后，至少审查：实际收集字段与目的、儿童/未成年人场景、同意与监护机制、保存期限、查看/修改/删除能力、第三方与跨设备流转、自动化画像，以及公开隐私说明与实际功能是否一致。

审查未完成时，只能声称“本地优先、默认不上传”等经测试的产品事实，不得声称“完全合规”“无需评估”或“绝对安全”。
