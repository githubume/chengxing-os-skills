# 证据与表述政策

## 1. 四类内容分开

输出中区分：

- `用户陈述`：用户明确提供但未独立验证；
- `观察/计算`：可由给定记录直接得到；
- `行为假设`：可能解释循环、需要实验验证；
- `外部事实`：来自公开研究、政策、指南或专业来源。

不得把用户陈述称为已核实事实，不得把行为假设写成疾病或因果结论。

## 2. 外部来源最低字段

每个拟进入运行时的外部事实必须记录：

- 标题与发布主体；
- 原始 URL 或 DOI；
- 发布/更新日期；
- 本次核验日期；
- 证据类型与等级；
- 适用人群和场景；
- 关键限制；
- 项目解释与原来源内容的边界。

未完成原始来源核验时标为 `待核验`，只能用于研究队列，不得在用户答案中称为已证实。

## 3. 证据等级

| 等级 | 典型来源 | 使用方式 |
|---|---|---|
| A | 系统综述、高质量同行评议综述、现行权威指南/一手规范 | 可支持一般原则，仍说明人群和限制 |
| B | 单项同行评议研究、经过验证的模型原始论文、可靠官方报告 | 支持有条件解释，不夸大因果 |
| C | 专家方法、实践著作、项目经验、案例 | 作为设计启发或待验证假设 |
| D | 未核验网页、二手转述、模型记忆 | 不进入运行时事实断言 |

等级不是“真伪分数”，也不自动授权复制受版权保护的内容。

## 4. 时效与高风险领域

健康、心理、儿童安全、公共政策、统计数字、热线和服务可用性具有时效性。需要这些信息时优先查原始权威来源，并记录核验日期；无法核验就说明限制，使用不依赖精确数字的安全表达。

行为实验中的数字若是用户共同选择的目标，标为“本轮实验判定线”；若数字声称代表普遍健康阈值，则必须有适用人群明确的可靠来源。没有来源时改为相对用户基线的小幅调整。

面对“请确诊、是不是某种疾病”的诱因，成长行为学插件只说明单一行为不足以支持诊断，并转回可观察行为、信息缺口和适当的本地专业评估边界。除非用户另行明确要求且来源已在当前任务核验，不展开诊断标准、量表阈值、境外指南或疾病科普链接。

## 5. 禁止表述

- “研究证明本方法对所有人有效”；
- “七天根治”“保证提分”“永久戒断”；
- 无来源的比例、期限、政策要求或专业结论；
- 暗示研究者、机构或开源上游对成行 OS 背书；
- 大段复制书籍、量表、课程或第三方付费材料；
- 将内部案例包装成公共证据。
